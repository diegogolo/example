Lista de requisitos funcionales​

A continuación se listan los requisitos identificados durante la toma de requerimientos, basados en las necesidades y problemas expresados por el cliente.

IDRequisitoDescripciónPrioridad (según el cliente)
RQ-01Pagos automatizadosEl sistema debe permitir a los clientes pagar con tarjeta de crédito, débito o transferencia electrónica, sin necesidad de que el dueño o empleados manejen efectivo manualmente.Alta
RQ-02Manejo de clientesEl sistema debe registrar y almacenar información de los clientes (nombre, teléfono, correo, historial de compras) para ofrecer un mejor servicio.Alta
RQ-03Manejo de empleadosEl sistema debe permitir registrar a los empleados, asignarles roles (cajero, administrador, etc.) y controlar su acceso al sistema.Media
RQ-04Manejo de inventarioEl sistema debe llevar el control de productos disponibles, entradas y salidas de mercancía, y alertar cuando un producto esté por agotarse.Alta
RQ-05Interfaz de pedidos para usuariosLos clientes deben poder realizar pedidos desde una interfaz sencilla (puede ser web o app) sin necesidad de hablar con un empleado.Media
RQ-06Automatización de tareas cotidianasEl sistema debe automatizar tareas repetitivas como generar facturas, actualizar inventario al vender, o enviar recordatorios a clientes.Alta
RQ-07Generación de resúmenes de actividades lucrativas por díaEl sistema debe producir un reporte diario que muestre ingresos, egresos, ganancias netas y número de transacciones realizadas.Alta
RQ-08Control de horarios de los empleadosEl sistema debe permitir registrar la hora de entrada y salida de cada empleado, calcular horas trabajadas y generar reportes de asistencia.Media

Clasificación por módulos del sistema​

Para facilitar el desarrollo y la negociación técnica, los requisitos se agrupan en los siguientes módulos:

Módulo de Ventas y Pagos​

  • RQ-01: Pagos automatizados
  • RQ-05: Interfaz de pedidos para usuarios

Módulo de Gestión de Usuarios​

  • RQ-02: Manejo de clientes
  • RQ-03: Manejo de empleados
  • RQ-08: Control de horarios de los empleados

Módulo de Inventario​

  • RQ-04: Manejo de inventario

Módulo de Reportes y Automatización​

  • RQ-06: Automatización de tareas cotidianas
  • RQ-07: Generación de resúmenes de actividades lucrativas por día

Notas adicionales​

  • Los requisitos con prioridad Alta deben estar sí o sí en la primera versión del sistema.
  • Los requisitos con prioridad Media pueden moverse a una versión 2.0 si el tiempo o presupuesto lo exigen.
  • El cliente mostró especial interés en RQ-01 (Pagos automatizados) y RQ-07 (Resúmenes diarios)

Requisitos no funcionales (identificados durante la entrevista)​

Además de los requisitos funcionales, se identificaron las siguientes expectativas del cliente sobre cómo debe comportarse el sistema:

IDTipoDescripción
RNF-01SeguridadLos pagos automatizados deben ser seguros y cumplir con estándares (PCI DSS).
RNF-02UsabilidadLa interfaz para clientes debe ser tan sencilla como usar una red social.
RNF-03DisponibilidadEl sistema debe estar funcionando siempre en horario de negocio (8 am a 8 pm).
RNF-04EscalabilidadDebe poder crecer si el negocio abre más sucursales.
